# Security Policy

## Supported versions

All packages at v0.x are pre-stable. Critical security fixes are backported only to the most recent v0.y release of each affected package. Once any package reaches v1.0, the previous major version receives security fixes for 12 months.

## Reporting a vulnerability

Please **do not** open a public issue for security reports. Email the project lead privately:

> _(set in Task 5 — replace with security@<your-domain>)_

We will acknowledge within 3 business days and aim to publish a fix within 90 days for non-critical issues, expedited if user-impacting.

## Disclosure timeline

- Day 0: report received, acknowledged within 3 business days.
- Day 1–60: investigation, fix, coordinated disclosure.
- Day 90: public disclosure (or earlier by mutual agreement).
