# Contributing to bare-swift

Thanks for your interest. A few ground rules before you open a PR.

## Conventions

Every package in this ecosystem follows [RFC-0001 — API Conventions](./rfcs/0001-api-conventions.md). Skim it before designing new public API.

## Discussions

For questions, ideas, and broad design discussions: GitHub Discussions on this repo. For a bug or specific feature request on a single package: open an issue on that package's repo.

## Pull requests

- Small, focused changes. One topic per PR.
- Tests required for any behavior change.
- All public API changes must include DocC updates.
- New conventions or anything that crosses package boundaries needs an RFC first (see [`rfcs/template.md`](./rfcs/template.md)).
- We aim to give a first response on a PR within 5 business days. A merge can take longer.

## Sign-off

We do not require a CLA. By submitting a PR you agree your contribution is licensed under Apache 2.0 with LLVM exception.

## Becoming a maintainer

A contributor who has landed 3+ non-trivial PRs over 3 months on a given package may be added as a co-maintainer of that package. See [MAINTAINERS.md](./MAINTAINERS.md).
